📊 PakOS Topic Dashboard

Visual management interface for all Telegram topics in PakOS ecosystem.

What is it?

A real-time dashboard that gives you a bird's-eye view of all active topics with:

  • Multiple views: Card, List, Kanban layouts
  • Topic details: Issues, memory, transcripts, skills, files
  • Health monitoring: System metrics, service status
  • Issue tracking: Edit and manage issues per topic

Key Features

  • Card View: Topics as cards with progress, status, metadata
  • List View: Sortable table format with quick scan
  • Kanban View: Topics organized by workflow status
  • Search: Filter topics by name in real-time
  • Detailed View: Deep dive into individual topic data

🔌 API Reference

All endpoints are prefixed with /api and return JSON.

Topics

GET /api/topics

List all active topics with summary data.

$ curl https://agents.pakhchau.com/api/topics

[
  {
    "id": "topic-24768",
    "name": "🗂️ Topic Dashboard",
    "topic_id": "24768",
    "status": "active",
    "heartbeat": "30m",
    "progress": 0,
    "has_issues": true,
    "has_memory": true
  },
  ...
]

GET /api/topics/{id}

Get detailed information about a specific topic.

$ curl https://agents.pakhchau.com/api/topics/topic-24768

{
  "id": "topic-24768",
  "name": "🗂️ Topic Dashboard",
  "overview": "...",
  "issues": [...],
  "memory": [...],
  "chat": [...],
  "skills": [...],
  "files": [...]
}

PUT /api/topics/{id}/issues

Update issues for a topic.

$ curl -X PUT https://agents.pakhchau.com/api/topics/topic-24768/issues \
  -H "Content-Type: application/json" \
  -d '{"issues": [{"title": "...", "done": false}]}'

Health

GET /api/health

System health status and metrics.

$ curl https://agents.pakhchau.com/api/health

🎨 Dashboard Views

Card View

Topics displayed as cards in a grid. Best for quick overview.

  • Shows: name, status, progress bar, heartbeat, icons
  • Click card to see details
  • Hover for subtle animations

List View

Compact table format. Best for scanning many topics.

  • Columns: Topic, Status, Progress, Heartbeat, ID
  • Sortable by column (click header)
  • Quick status badges

Kanban View

Topics organized by workflow status. Best for project tracking.

  • Columns: Active, Paused, Dormant, Completed
  • Drag to move between statuses (coming soon)
  • Count indicators per column

Detail View

Click any topic to open detailed view with tabs:

  • Overview (📊): Description, metadata, stats
  • Issues (📋): Open/done issues, add/edit
  • Memory (🧠): Session notes and context
  • Chat (💬): Message transcript
  • Skills (⚡): Available capabilities
  • Files (📁): Outputs and artifacts
  • Crons (⏰): Scheduled tasks and heartbeats

📊 Data Sources

Where Information Comes From

Topic Metadata

From /root/clawd/topics/ workspace directories:

  • SOUL.md — Topic name, role, mission
  • ISSUES.md — Issue list (editable)
  • HEARTBEAT.md — Heartbeat schedule
  • SKILLS.md — Available capabilities

Memory

From /root/clawd/topics/*/memory/:

  • Daily session notes (YYYY-MM-DD.md)
  • Long-term memory (MEMORY.md)

Chat Transcripts

From /root/.openclaw/agents/*/sessions/*.jsonl:

  • Session messages with timestamps
  • User and assistant responses
  • Searchable and filterable

System Health

From system diagnostics:

  • Disk usage, memory, CPU
  • Service status
  • Open issues count

🚀 Deployment

Production Setup

This dashboard is hosted at agents.pakhchau.com and served by:

  • Backend: /root/clawd/pakos-dashboard/server.py
  • Port: 8935 (proxied via nginx)
  • Frontend: Static HTML/CSS/JS

Updating the Dashboard

Edit files in /root/clawd/pakos-dashboard/:

$ cd /root/clawd/pakos-dashboard/
$ # Edit index.html, server.py, etc.
$ # Restart service
$ systemctl restart pakos-dashboard

Adding New Pages

Create new HTML files and wire them in server.py:

if path == '/docs':
    return serve_file('docs.html')
elif path == '/test':
    return serve_file('test.html')
elif path == '/newpage':
    return serve_file('newpage.html')

API Endpoints

Add new routes in server.py request handler.

❓ FAQ

How do I add a topic?

Topics are discovered automatically from /root/clawd/topics/ folders. Create a new folder with SOUL.md and it will appear in the dashboard.

Can I edit issues?

Yes! Click a topic to open the detail view, go to the Issues tab, and use the edit buttons to add, modify, or remove issues.

Where does the memory data come from?

From memory/*.md files in each topic's workspace, plus MEMORY.md for long-term context.

How often does data refresh?

Data is fetched on page load and when you click "Refresh". Real-time updates coming soon.

What if a topic doesn't show data?

Make sure the topic folder exists at /root/clawd/topics/topic-XXXX/ and has appropriate files (SOUL.md, ISSUES.md, etc.).

Can I search topics?

Yes! Use the search box at the top of the dashboard. It filters by topic name in real-time.

How do I change the view?

Click "Cards", "List", or "Kanban" buttons at the top right of the dashboard.

What's the Heartbeat column?

Shows how often the topic checks for new messages. Examples: "30m" (every 30 minutes), "1h" (hourly), "off" (disabled).

How do I see chat transcripts?

Open a topic detail view and click the "Chat" tab to see message history.

Can I export data?

Not yet, but it's planned. All data is readable via the /api endpoints.
